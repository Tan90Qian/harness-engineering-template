# 文档-代码同步映射表（SYNC_MAP）

> **用途**：声明代码文件/目录与文档的对应关系。
> 当代码变更时，CI 和 AI 可据此检测对应文档是否需要同步更新。

## 映射规则

```
代码路径 (glob)  →  文档路径  →  说明
```

<!-- 同步级别：strict = CI 报错 / warn = CI 警告 / info = 仅提醒 -->

| 代码路径 | 对应文档 | 同步级别 | 说明 |
|----------|---------|---------|------|
| `src/types/**/*.ts` | `docs/design/design-tokens.md` | warn | 类型变更可能影响设计规范 |
| `src/pages/**/*` | `docs/pages/*.spec.md` | strict | 页面代码必须有对应 Page Spec |
| `src/api/**/*` | `.ai-rules.md` | info | API 变更需检查 AI 规则是否过期 |

## 如何添加新映射

在上表中添加一行即可。`scripts/doc-sync-check.js` 会自动解析此表。

## 同步级别说明

- **strict**：代码变了但文档没变 → CI **报错**，阻止合并
- **warn**：代码变了但文档没变 → CI **警告**，不阻止但会在 PR 中提示
- **info**：仅在开发时由 AI 提醒，CI 不做检查

## 最佳实践

1. **新增功能时**：先添加映射，再写代码
2. **紧急修复时**：允许跳过（CI warn 级别），但必须在下一个 Sprint 补文档
3. **重构时**：同步更新所有 strict 映射的文档
4. **删除功能时**：同步删除映射和文档
