## 触发条件

当开发者要求生成页面、组件、或包含接口调用的代码时自动触发。

## 检查步骤

### Step 1: 确定目标
从开发者的请求中识别：
- 目标模块名
- 目标页面名
- 目标技术栈（如有多端）

### Step 2: 检查 Page Spec
查找：`docs/pages/{module}/{page}.spec.md`
- 存在 → 读取并理解
- 不存在 → 提醒并提供模板

### Step 3: 检查接口类型
查找项目中对应模块的类型定义文件
- 存在 → 读取类型定义
- 不存在 → 提醒并提供模板

### Step 4: 检查设计规范
查找：`docs/design/design-tokens.md`
- 存在 → 读取
- 不存在 → 使用默认值并告知

### Step 5: 检查状态机（如涉及）
查找对应模块的状态定义文件
- 不存在 → 提醒并提供模板

### Step 6: 汇总报告
输出上下文检查报告，列出 ✅ / ⚠️ / ❌ 状态。
全部就绪 → 开始生成代码。
有缺失 → 按引导行为矩阵处理。

## 引导行为矩阵

| 缺失项 | AI 行为 | 提供的帮助 |
|--------|---------|-----------|
| Page Spec 不存在 | 🛑 暂停生成，提醒 | 提供 spec 模板，引导填写 |
| 接口类型不存在 | 🛑 暂停生成，提醒 | 提供类型定义模板 |
| 设计规范不存在 | ⚠️ 警告但可继续 | 使用默认值，列出默认值 |
| 状态机未定义 | 🛑 暂停生成，提醒 | 提供状态机模板 |
| 开发者描述模糊 | 🛑 先澄清再做 | 给出 2-3 个选项让开发者选择 |
