// turbo
1. **上下文检查**
   - 读取 `docs/pages/{module}/{page}.spec.md` → 不存在则引导创建
   - 读取该模块的类型定义 → 不存在则引导创建
   - 读取 `docs/design/design-tokens.md` → 不存在则使用默认值

2. **上下文完整 → 直接生成；不完整 → 先引导补全再生成**

3. **生成页面文件**
   - 遵循项目 UI 框架约定
   - 从共享类型包中导入类型
   - 处理加载中/错误/空数据状态

4. **更新路由配置**

5. **文档同步**（强制）
   - 确认 `docs/pages/` 下对应 Page Spec 已创建/更新
   - 检查 `docs/SYNC_MAP.md`，必要时添加新映射
