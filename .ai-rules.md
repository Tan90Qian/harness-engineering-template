## 通用规则

- 遵循 TypeScript strict 模式
- 提交代码遵循 Conventional Commits 规范：Header 英文 + Body 中文（可通过 `git-commit` skill 生成）
- 提交 scope 取值：`all` | `core`（按项目实际 scope 修改）
- 提交必须包含中文 body，用 `- ` 列表描述具体改动（What / Why / How）
- 环境变量通过 `.env` 管理，敏感信息不得硬编码

## 代码质量

- 所有共享类型必须从统一入口导出
- 优先组合而非继承
- 禁止使用 `any`，除非附带注释说明原因
- 所有公开函数必须有 JSDoc 注释
- 错误处理必须显式，禁止静默吞掉异常

## 测试规则

- 工具函数必须有单元测试
- API 接口必须有集成测试
- 核心用户流程应有 E2E 测试
- 测试工具：Vitest（单元/集成）+ Playwright（E2E）

## AI Skills（Skill-only）

### Skill 调用规则

**重要**：本模板统一使用 Skill 机制，不保留 Workflow 入口。

当用户提出开发/修复/文档同步相关需求时，AI 必须：

1. **识别任务类型**：映射到对应 Skill
2. **读取 Skill**：检查 `.windsurf/skills/{skill-name}/SKILL.md`
3. **按 Skill 执行**：严格按步骤执行，不猜测业务细节
4. **先检查后生成**：涉及代码生成时先执行 `check-context`

### 最小可落地 Skill 列表（默认基线）

| Skill | 用途 |
| --- | --- |
| `check-context` | 开发前上下文完整性检查 |
| `create-type-definition` | 创建/更新 TypeScript 类型定义 |
| `create-page-spec` | 创建页面 Spec |
| `create-component-spec` | 创建组件 Spec |
| `create-react-page` | 生成 React 页面 |
| `create-uni-page` | 生成 uni-app 页面 |
| `create-bff-module` | 生成 BFF 模块 |
| `complete-implementation` | 对照 Spec 补全实现 |
| `fix-bug` | 定位根因并最小化修复 |
| `health-check` | 运行 AI 工程化健壮性检查 |
