## 通用规则

- **始终使用中文回复** - 所有对话、代码注释、文档都必须使用中文
- 遵循 TypeScript strict 模式
- 提交代码遵循 Conventional Commits 规范：Header 英文 + Body 中文（详见 `/git-commit` workflow）
- 提交 scope 取值：`all` | `core`（按项目实际 scope 修改）
- 提交必须包含中文 body，用 `- ` 列表描述具体改动（What / Why / How）
- 环境变量通过 `.env` 管理，敏感信息不得硬编码

## 代码质量

- 所有共享类型必须从统一入口导出
- 优先组合而非继承
- 禁止使用 `any`，除非附带注释说明原因
- 所有公开函数必须有 JSDoc 注释
- 错误处理必须显式，禁止静默吞掉异常

## 测试规则

- 工具函数必须有单元测试
- API 接口必须有集成测试
- 核心用户流程应有 E2E 测试
- 测试工具：Vitest（单元/集成）+ Playwright（E2E）

## AI 工作流

### Workflow 指令处理规则

**重要**：当用户输入以 `/` 开头的指令时，AI 必须：

1. **识别指令**：检查 `.windsurf/workflows/` 目录下是否存在对应的 `.md` 文件
2. **读取 workflow**：使用 `read_file` 工具读取对应的 workflow 文件
3. **执行步骤**：按照 workflow 中定义的步骤逐步执行
4. **遵循 turbo 标记**：标记为 `// turbo` 的步骤可以自动执行

### 可用 Workflow 列表

| 命令 | 用途 |
| --- | --- |
| `/new-feature` | 开发新功能（从 spec 到测试） |
| `/new-page` | 开发新页面 |
| `/fix-bug` | 修复 Bug |
| `/git-commit` | 编写规范的 git commit 信息（英文 Header + 中文 Body） |
| `/complete` | 完善现有实现（对比 spec 文档补全缺失功能） |
| `/health-check` | 运行 AI 工程化健壮性自动检查（评分 + 待改进项） |
| `/doc-workflow` | 文档工作流 - 支持单人/团队模式切换 |
| `/switch-to-team-mode` | 一键切换项目到团队模式（不可逆） |
